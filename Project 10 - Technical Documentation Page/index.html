What Is PostgreSQL

PostgreSQL ia a powerful, open source object-relational database system with over 35 years of active development.

The ability of PostgreSQL to support both SQL (relational) and JSON (non-relational) querying has earned it a strong reputation for reliability, data integrity, feature robustness, and performance.

Hence its no surprise that PostgreSQL has become the open-source relational database running on all major operating systems of choice for many people and organisations.

Below is an inexhaustive list of various modern features offered by PostgreSQL:
  • complex queries
  • foreign keys
  • triggers
  • updatable views
  • transactional integrity
  • multi-version concurrency control

Furthermore, PostgreSQL is highly extensible.

For example, a user can define their own data types, functions, operators, index methods, write code from different programming languages without recompiling the database etc.

Getting Started with PostgreSQL

This section helps you get started with PostgreSQL on Linux by achieving the following:

  1. show you how to install PostgreSQL on Linux,
  2. connect to PostgreSQL using the psql tool,
  3. load a sample database into the PostgreSQL for practising

Install PostgreSQL Linux

Most Linux platforms such as Debian, Red Hat/ CentOS, SUSE, and Ubuntu have PostgreSQL integrated with their package management.

It is recommended that you install PostgreSQL this way since it ensures a proper integration with the operating system including automatic patching and other upgrade management functionality.

To download and install PostgreSQL on Linux, visit the download page and select your Linux distribution.

Once on the Ubuntu option page, execute the below commands in the order provided:

  1. create the file repository configuration $ sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
  2. import the repository signing key: $ wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
  3. update the package list $ sudo apt-get update
  4. install the latest version of PostgreSQL $ sudo apt-get install postgresql

Connect to the PostgreSQL database server via psql

In PostgreSQL, a user account is referred to as a role

By default, PostgreSQL uses ident authentication

It means that PostgreSQL will associate its roles with the system accounts of Linux.

If a role exists in PostgreSQL, the same Linux user account with the same name is able to log is as that role

When you installed PostgreSQL the installation process created a user account called postgres associated with the default postgres role.

To connect to PostgreSQL using the postgres role, you switch over to the postgres account on your server by following the following commands:

  1. $ sudo -i -u postgres

    Enter your account user name and hit Enter

  2. Access the PostgreSQL using the psql by typing the below command: $ psql
  3. You'll access the postgres prompt like this: postgres=#

    From her you can interact with the PostgreSQLlike using a query.

  4. To quit PostgreSQL, run the below command: postgres=# \q
  5. To exit, run the below command: exit

Load the sample database

To load the sample database into the PostgreSQL database server, follow the below steps:

  1. $ sudo -i -u postgres

    Enter your password and hit Enter

  2. download the sample database using curl tool. $ curl -O https://sp.postgresqltutorial.com/wp-content/uploads/2019/05/dvdrental.zip
  3. unzip the dvdrental.zip file to get the dvdrental.tar file: $ unzip dvdrental.zip
  4. access the PostgreSQL using the psql tool: $ psql
  5. create the dvdrental database using the CREATE DATABASE statement: postgres=# create database dvdrental;
  6. quit the psql by using the \q command: postgres=# \q
  7. us the pg_restore tool to restore the dvdrental: $ pg_restore --dbname=dvdrental --verbose dvdrental.tar
  8. access PostgreSQL database server again using psql: $ psql
  9. switch to dvdrental database: postgres=# \c dvdrental

    Now, you;re connected to the dvdrental database:

    dvdrental=#
  10. Finally, enter the following command to get the number of films in the film table: dvdrental=# select count(*) from film;

    Here is the output:

    count ------- 1000 (1 row)

    Congratulations! you have successfully installed PostgreSQL on Ubuntu, connect to PostgreSQL database server using psql, and load the sample database.

PostgreSQL Views

A view is a named query that provides another way to present data in the database

It is defined based on the on one or more tables which are known as data tables

Views are useful for wrapping a commonly used complex query

In this section, you will learn about the PostgreSQL views concept and how to manage views such as creating, modifying, and removing views from the database.

NB* A view does not physically store data, however you can query a view using the SELECTstatement.

Creating PostgreSQL Views

To create a view, we use CREATE VIEWstatement, syntax below:

CREATE VIEW view_name AS query;

Changing PostgreSQL Views

To change the defining query of a view, you use the CREATE VIEWstatement with OR REPLACE addition as follows:

CREATE OR REPLACE view_name AS query

To change the definition of a view, you use the ALTER VIEW statement.

For example, to change the name of the view from customer_master to customer_info by using the following statement:

ALTER VIEW customer_master RENAME TO customer_info;

PostgreSQL allows you to set a default value for a column name, change the view's schema, set or reset option of a view.

For detailed information on altering view;s definition, click here

Removing PostgreSQL Views

To remove existing view in PostgreSQL, you use the DROP VIEW statement, see syntax below:

DROP VIEW [ IF EXISTS ] view_name;

Note, the IF EXIST option to the statement ensures that the query does not throw an err should you try to remove a view that does not exist in the database.

It therefore instruct PostgreSQL to remove the view if it exists, otherwise do nothing.

PostgreSQL Indexes

These are effective tools to enhance database performance.

Indexes help the database server find specific rows much faster than it could do without them

Indexes add write and storage overheads to the database system, hence using them appropriately is of paramount importance.

  • Partial Index - illustrates how to use partial indexes

PostgreSQL Triggers

A PostgreSQL trigger is a function invoked automatically whenever an event such as insert, update, delete occurs.
